Home

  • For most of the page elements, when hovered over, they will change color to indicate they are clickable for interaction. They will load to a new page. This was achieved using the .mouseover and .mouseout event handlers and Javascript functions to change the color attributes.
  • All elements were created using Snap.SVG.

Worm Page

  • When clicked, the worm moves off the screen to the right, looking like it moves through the tunnel and out of the window. This was achieved by creating Javascript functions that utilize the .animate handler of Snap.SVG and appending and calling the function to the graphics when they are clicked.
  • When one clicks the stem, the worm moves back into the screen and returns to his original position utilizing the same functionality as when the worm is clicked.
  • Both the stem and the worm body change color when hovered over to show they are interactive and can be clicked on.

Flower Page

  • Clicking on the center of the flower calls Javascript function that uses the .animate function of Snap.SVG to move the bee up and out of the SVG window, almost like he is flying away.
  • The stem can also be clicked on to return the bee to his original position in the window, and the two functions can be toggled to move the bee in and out.
  • The center of the flower and the stem shange color when hovered over to show interactivity.

Tree Page

  • The apples on the tree may be dragged around the screen and places wherever one would like as they do not follow the rules of gravity. The draggability is achieved using jQuery.
  • If, at anytime, the user decides they do not like where the apples are, they can click the trunk of the tree to move the apples back to their original position in the window. Once again, this is achieved by using the .animate function of Snap.SVG contained in a function that focuses on transforming the x and y positions of whatever element is specified.
  • The trunk and the apples change color when hovered over to show interactivity.

Above and Beyond

  • All images and content were created by myself, unless otherwise stated above. That means, every single graphic was coded by myself using Snap.SVG. Creating the SVGs was definitely the most tedious, challenging, and time-consuming part of this project for me. I taught myself how to use this Javascript library over the course of this project because it seemed like an interesting way to create SVGs without using Illustrator or Photoshop.
  • I used topics not discussed in class to animate the graphics, like complex .animate functions that focus on transforms. I also used Snap.SVG
  • I chose to keep the HTML and CSS fairly simple so that I could really focus on the SVGs and Javascript functionality.
